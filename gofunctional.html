Functional Go Code: First Attempt!

Why?

Except for the usual reason because I can,but also because I wanted to learn how one would do functional style programming similar to what is supported by Javascript in Go.

The Idea

Go supports closures and first class functions, so in that regard we're good to ..err.. continue. I've already implemented a library of functional helpers in Lua, based on the javascript methods:

  • forEach,
  • reduce and
  • map

I will use forEach as the basis of my functional util library. Reduce and map will both call forEach to handle iteration. (Probably not the right way to do it, but fixing that will be an excersize for later.)

Limitations

  • My Lack of Go experience! The main issue here!
  • Go's Type System
  • No tail recursion optimization. It just means I have to limit recursion tricks

The Complete Code

Currently I'm only implementing the code to work on a very specific usecase: To be able to act on a float64 array. The next step will be to find some way of making it generic. Perhaps interfaces will help?

Stuff I've learnt

  • Always remember to declare the function type and add a return! (Otherwise you get told "Too many values")
  • Making a functional util library may be a bit tricky to the beginner Go-pher.
  • Using interface{} may help as used by this guy. Though switching to another array type is still slightly tricky.
